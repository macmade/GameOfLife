Roadmap

  1. Unit Tests

    Introduce a unit-test target to the project, which currently has none, and build out broad coverage over time. Start with the simulation core — grid evolution, rule application, and the RLE/cell readers and writers — then extend to parsing, the pattern library, and supporting utilities. The goal is a safety net that guards correctness as the app evolves.

  2. Infinite Grid

    Replace the current fixed-size grid (a bounded width × height array) with an unbounded grid, allowing the simulation to grow without edges or boundaries. Patterns such as gliders and guns can then run indefinitely without hitting the limits of a fixed playfield.

  3. Modernize Swift

    Bring the codebase up to idiomatic, modern Swift. Favour functional, declarative constructs — map, filter, reduce, forEach — over raw for loops, and adopt current language features and conventions throughout. The aim is clearer, more expressive, and more maintainable code.

  4. Documentation Comments

    Add Swift documentation comments (the /// DocC markup) across the codebase, covering public and private types and members alike. This documents intent directly in the source, surfaces it via Xcode’s Quick Help, and lays the groundwork for generating a browsable DocC reference.
